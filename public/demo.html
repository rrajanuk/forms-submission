🎯 Form Embed SDK Demo

1. Create a Test Form First

Before testing the embed, you need to create a published form. Run these commands:

# 1. Register a user and organization curl -X POST http://localhost:3001/api/auth/register \ -H "Content-Type: application/json" \ -d '{"name":"Demo Org","email":"demo@example.com","password":"demo123"}' # 2. Login to get token curl -X POST http://localhost:3001/api/auth/login \ -H "Content-Type: application/json" \ -d '{"email":"demo@example.com","password":"demo123"}' # 3. Create a form (replace YOUR_TOKEN) curl -X POST http://localhost:3001/api/forms \ -H "Authorization: Bearer YOUR_TOKEN" \ -H "Content-Type: application/json" \ -d '{ "name": "Customer Feedback", "slug": "customer-feedback", "description": "Share your thoughts with us", "schema": { "fields": [ { "id": "name", "type": "short_text", "label": "What is your name?", "required": true }, { "id": "email", "type": "email", "label": "What is your email?", "required": true }, { "id": "rating", "type": "rating", "label": "How would you rate us?", "required": true, "validation": { "max": 5 } }, { "id": "feedback", "type": "long_text", "label": "Any additional feedback?", "required": false } ], "theme": { "layout": "classic", "primaryColor": "#4F46E5" } } }' # 4. Publish the form curl -X POST http://localhost:3001/api/forms/FORM_ID/publish \ -H "Authorization: Bearer YOUR_TOKEN"

2. Embed Code Example

Use this code to embed the form on your website:

<!-- Add this to your HTML <head> or before </body> --> <link rel="stylesheet" href="http://localhost:3001/embed/embed.css"> <script src="http://localhost:3001/embed/embed.js"></script> <!-- Add this where you want the form to appear --> <div id="my-form"></div> <script> new FormEmbed({ formSlug: 'customer-feedback', // Change to your form's slug container: document.getElementById('my-form'), theme: { primaryColor: '#4F46E5', backgroundColor: '#ffffff', layout: 'classic' }, onProgress: (data) => { console.log('Form progress:', data); }, onComplete: (response) => { console.log('Form submitted!', response); alert('Thank you for your feedback!'); }, onError: (error) => { console.error('Form error:', error); } }); </script>
Note: For production, replace http://localhost:3001 with your actual domain.

3. Live Test

Once you've created and published a form with slug customer-feedback, the form will appear below:

Features Demo

  • ✅ Auto-save: Draft is saved every 30 seconds or 2 seconds after changes
  • ✅ Session persistence: Refresh the page to restore your draft
  • ✅ Validation: Real-time field validation
  • ✅ Progress tracking: Shows completion progress
  • ✅ Responsive: Works on mobile and desktop
  • ✅ Customizable: Change colors, layout, and styling

4. Test Auto-Save

  1. Start filling out the form above
  2. Wait 30 seconds (or make changes and wait 2 seconds)
  3. Refresh the page
  4. Your draft should be automatically restored!
To check drafts via API:
curl http://localhost:3001/api/public/forms/customer-feedback/draft/YOUR_SESSION_ID
The session ID is logged to the browser console when the form loads.
